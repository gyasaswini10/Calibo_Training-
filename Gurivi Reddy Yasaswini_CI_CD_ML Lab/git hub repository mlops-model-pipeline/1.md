

Objective

The aim of this assignment is to understand how a machine learning model works in a real-world system, not just training it once.

We are building a pipeline that can:
• Train a model  
• Check if it is good  
• Deploy it safely  
• Monitor its performance  
• Retrain when needed


Overall Flow

First, data is taken and a model is trained.
After training, the model is tested using validation rules to make sure it is performing well.  

If the model passes all checks, it goes into deployment using CI/CD.  
Initially, it is released to a small number of users (canary deployment).  
If everything works fine, it is fully deployed.  

After deployment, the system keeps monitoring the model.  
If accuracy drops or data changes, retraining is triggered.  


Steps

1. Train model → train.py  
2. Validate model → validate.py  
3. Run CI pipeline → ci_pipeline.yml  
4. Deploy model → cd_deploy.yml  
5. Monitor performance → monitor.py  
6. Retrain if needed

My Understanding

In real life, models are not used just after training once.  
Data keeps changing, so model performance can decrease.  

Because of that:  
• We validate models before using them  
• We deploy them slowly to avoid risk  
• We monitor them continuously  
• We retrain them automatically  

This makes the system reliable and closer to real industry usage.
